## 三、数据库表结构

### 3.1 users 用户表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK AUTO_INCREMENT | 用户ID |
| phone | VARCHAR(20) UNIQUE | 手机号，登录唯一标识 |
| username | VARCHAR(50) | 用户名，2-30字符 |
| password_hash | VARCHAR(255) | bcrypt 哈希 |
| role | ENUM('candidate','admin') | 角色 |
| is_disabled | TINYINT(1) | 禁用标记 |
| created_at / updated_at | DATETIME | 时间戳 |

### 3.2 resumes 简历表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 简历ID |
| user_id | INT FK → users | 所属用户 |
| file_path | VARCHAR(500) | 文件路径 |
| parsed_data | JSON | 结构化数据（姓名/技能/经历等） |
| position | VARCHAR(100) | 目标岗位 |
| difficulty | ENUM('初级','中级','高级') | 难度 |
| position_match_score | INT | 岗位匹配度 0-100 |
| match_feedback | TEXT | 匹配度说明 |

### 3.3 interviews 面试表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 面试ID |
| user_id / resume_id | FK | 候选人 + 简历 |
| position / difficulty | — | 岗位 + 难度 |
| mode | ENUM('full','stage') | 全流程/阶段练习 |
| status | ENUM('created','in_progress','completed','abandoned') | 状态 |
| answers | JSON | 所有问答对（含评分/错误/录音URL） |
| overall_score / passed | INT / TINYINT | 总分 + 是否通过 |
| started_at / ended_at | DATETIME | 时间戳 |

### 3.4 reports 报告表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 报告ID |
| interview_id | INT FK UNIQUE | 关联面试 |
| report_data | JSON | 完整报告（维度分/阶段分解/错误解析/简历扣分） |
| status | ENUM('generating','completed','error') | 生成状态 |

### 3.5 questions 题库表

| 字段 | 类型 | 说明 |
|------|------|------|
| stage | VARCHAR(50) | 阶段（初筛/HR面/技术面/终面） |
| type | ENUM('technical','behavioral','situational') | 题型 |
| position_tags | JSON | 适用岗位标签 |
| dimensions | JSON | 考察维度 |
| scoring_points | JSON | 评分要点（含max_score） |
| follow_up_hints | JSON | 追问方向 |
| is_deleted | TINYINT(1) | 软删除 |

### 3.6 documents 文档表

| 字段 | 类型 | 说明 |
|------|------|------|
| status | ENUM('processing','ready','error') | 向量化状态 |
| chunks_count | INT | 切片数量 |

### 实体关系

```
users 1──N resumes
users 1──N interviews 1──1 reports
                   └── answers JSON（内嵌问答对）
questions（平台题库，软删除）
documents（非结构化文档，FAISS 索引）
feedbacks（举报纠正记录）
```

## 四、API 接口速查

**基础URL：** `https://域名`  
**认证：** `Authorization: Bearer {access_token}`  
**WebSocket：** `wss://域名/ws/interview/{id}?token={ws_token}`

### REST API（18 个接口）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/register` | 注册（邀请码+短信+密码） |
| POST | `/api/auth/login` | 登录，返回 JWT 对 |
| POST | `/api/auth/refresh` | 刷新令牌 |
| POST | `/api/auth/logout` | 登出（吊销 token） |
| POST | `/api/auth/reset-password` | 短信验证码重置密码 |
| GET | `/api/auth/invite-code` | 获取邀请码（管理员） |
| POST | `/api/captcha/send` | 发送短信验证码 |
| POST | `/api/captcha/verify` | 验证短信验证码 |
| POST | `/api/resumes/upload` | 上传简历（multipart） |
| GET | `/api/resumes/{id}` | 简历详情 |
| POST | `/api/interviews` | 创建面试，获取 ws_token |
| GET | `/api/interviews/history` | 面试历史列表 |
| GET | `/api/interviews/{id}/report` | 面试报告 |
| POST | `/api/feedback/{id}` | 提交评分纠正反馈 |
| \* | `/api/admin/questions` | 题库 CRUD |
| \* | `/api/admin/documents` | 文档管理 |
| GET | `/api/admin/users` | 用户列表 |
| PUT | `/api/admin/users/{id}/disable` | 禁用/启用用户 |

### WebSocket 消息类型

**服务端→客户端：** `session/started` / `ai/text` / `ai/status` / `control/stage_change` / `control/coding_challenge` / `user/text_echo` / `session/end` / `pong`  
**客户端→服务端：** `user/speech_end` / `user/text` / `user/interrupt` / `control/submit_code` / `control/end_session` / `ping` + 二进制音频帧

### 错误码

| HTTP | code | 说明 |
|------|------|------|
| 401 | UNAUTHORIZED | 未登录或 token 无效 |
| 403 | FORBIDDEN | 权限不足 |
| 404 | NOT_FOUND | 资源不存在 |
| 409 | CONFLICT | 资源冲突（如手机号已注册） |
| 413 | FILE_TOO_LARGE | 文件过大 |
| 422 | VALIDATION_ERROR | 参数校验失败 |
| 429 | RATE_LIMITED | 请求过于频繁 |

## 五、AI Agent 体系

| Agent | 功能 | 调用时机 |
|-------|------|---------|
| ResumeAgent | 简历解析 + 岗位匹配评分 | 简历上传时 |
| InterviewerAgent | 四阶段差异化出题 + 追问决策 | 面试过程中 |
| ScoringAgent | 五维度实时评分（技术深度/广度/工程化/沟通/匹配度） | 每道题回答后 |
| CodeReviewAgent | 代码四维评审（正确性/性能/可读性/安全性） | 编程题提交时 |
| ReportAgent | 综合报告生成（含简历扣分 0-15） | 面试结束后 |

**四阶段设计：**

| 阶段 | 目的 | 题数 | 追问 |
|------|------|------|------|
| 初筛 | 验证技能基础 | 2-3 | 1-2轮 |
| HR面 | 评估性格协作 | 2-3 | 2-3轮 |
| 技术面 | 深挖技术栈 | 1-2 | 强制3-4层 |
| 终面 | 综合视野潜力 | 1-2 | 1-2轮 |

## 六、前端架构要点

### 路由设计

| 路径 | 页面 | 权限 |
|------|------|------|
| `/login` | LoginPage | 未登录 |
| `/register` | RegisterPage | 未登录 |
| `/forgot-password` | ForgotPasswordPage | 未登录 |
| `/dashboard` | DashboardPage | 已登录 |
| `/interview/:id` | InterviewRoom | 已登录 |
| `/report/:interviewId` | ReportPage | 已登录 |
| `/admin/questions` | QuestionManage | 管理员 |
| `/admin/documents` | DocumentUpload | 管理员 |

### 面试房间状态机

| 状态 | 麦克风 | AI 显示 |
|------|--------|---------|
| 聆听态 | 绿色脉动 | 静止 |
| 思考态 | 禁用灰色 | "AI正在思考..." |
| 说话态 | 禁用灰色 | 波形跳动 + TTS 播放 |
| 解析态 | 灰色+Loading | 等待 STT 结果 |

### PTT（按住说话）模式

1. 点击麦克风开始录音，松开后停止
2. STT 识别 → 草稿气泡（可编辑）
3. 4 秒后自动发送，编辑文字时取消自动发送
4. 支持重新回答

## 七、设计风格

- **调性：** 专业的对话记录仪，聚焦内容
- **色彩：**
  - 主背景 `#F7F8FA` / 卡片 `#FFFFFF`
  - 强调色 `#2B3A67`（藏蓝）
  - 成功 `#4A7C59`（墨绿）/ 警示 `#C27A3D`（橙红）
- **布局：** 1px 灰色边框分割，去除大阴影圆角，消息统一靠左
- **动效：** 200ms 以内，克制快速
- **伦理声明：** 固定在页面底部 Footer："此评分为模拟练习参考，不构成真实招聘决策依据"
