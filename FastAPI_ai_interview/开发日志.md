### Phase 2.2 — 文档向量化

**创建/修改的文件：**
- `app/services/vector_store.py` — FAISS 向量存储封装
- `app/api/v1/admin/documents.py` — 上传后同步触发向量化
- `requirements.txt` — 新增 `langchain`, `langchain-community`, `langchain-text-splitters`, `faiss-cpu`, `sentence-transformers`, `numpy`

**核心能力：**
- `chunk_text()` — LangChain RecursiveCharacterTextSplitter 分块，支持中英文分隔符
- `add_document()` — 分块→嵌入→FAISS IndexFlatIP 索引→磁盘持久化
- `search_all()` / `search_document()` — 向量相似度检索（余弦相似度，归一化向量 + 内积）
- `delete_document()` — 删除 FAISS 索引 + 元数据
- 嵌入模型：`BAAI/bge-small-zh-v1.5`（通过 sentence-transformers 异步调用）
- FAISS 索引持久化路径：`uploads/faiss/doc_{id}.faiss`

### Phase 2.3 — RAG 检索服务

**创建的文件：**
- `app/services/rag_service.py` — `RAGService` 类

**接口：**
- `search_questions(stage, position, difficulty, type, limit)` — SQL 题库检索，支持 position_tags 模糊匹配
- `search_documents(query, top_k)` — FAISS 向量文档检索
- `get_context_for_interview(query, stage, position, difficulty)` — 混合检索，一次返回题目+文档上下文
- `get_question_by_id(question_id)` — 单题查询

### Phase 2.4 — InterviewerAgent + CodeReviewAgent

**创建的文件：**
- `app/agents/interviewer_agent.py` — 面试官 Agent
- `app/agents/code_review_agent.py` — 代码评审 Agent

**InterviewerAgent 接口：**
- `start_stage(stage, position, difficulty, context)` — 开始新阶段，生成首个问题
- `evaluate_answer(stage, position, difficulty, question_text, user_answer, question_count, max_questions)` — 评估回答，返回 `ask_question`/`follow_up`/`stage_complete` 三态决策
- `generate_final_summary(position, stage_summaries)` — 生成面试结束语

**CodeReviewAgent 接口：**
- `review(code, language, question_title, question_description, difficulty)` — 四维评审（正确性/性能/可读性/安全性），返回结构化 JSON 含总分、优点、不足、改进代码、建议

### Phase 2.5 — 面试编排引擎

**创建的文件：**
- `app/services/interview_orchestrator.py` — `InterviewOrchestrator` 类

**状态机：**
```
created → in_progress → completed/abandoned
```

**核心流程：**
1. `start()` — 发送 `session/started`，更新状态为 `in_progress`
2. `begin_current_stage()` — 发送 `control/stage_change`，从题库→Agent 生成首题
3. `handle_text(text)` — 接收回答→存储→Agent评估→追问/下一题/阶段结束
4. `handle_code(code, language)` — 代码评审→返回结构化反馈
5. `handle_end()` — 用户主动结束，状态→`abandoned`
6. `finalize()` — 正常结束，创建 Report 占位，状态→`completed`

**设计决策：**
- 题目选择策略：优先从题库匹配（`RAGService.search_questions`），无匹配时由 `InterviewerAgent` 动态生成
- 阶段顺序：全流程模式固定 `[初筛, HR面, 技术面, 终面]`，阶段模式使用 `selected_stages`
- 每阶段最多 4 题（`MAX_QUESTIONS_PER_STAGE`）

### Phase 2.6 — WebSocket 实时通信（文字版）

**创建的文件：**
- `app/ws/interview_ws.py` — WebSocket 路由处理器
- `app/ws/session_manager.py` — 会话状态管理（Redis Hash）
- `app/ws/__init__.py` — 模块导出

**修改的文件：**
- `app/main.py` — 注册 WebSocket 端点 `/ws/interview/{interview_id}?token=...`
- `app/api/v1/interviews.py` — 创建面试时存储 ws_token 到 Redis

**支持的 WS 消息类型：**

服务端→客户端：
| 类型 | 说明 |
|------|------|
| `session/started` | 面试开始，携带 interview_id 和阶段信息 |
| `control/stage_change` | 阶段切换通知，含 from/to 和首个问题 |
| `ai/text` | AI 面试官文本回复（is_final: true） |
| `session/end` | 面试结束，含结束语 |

客户端→服务端：
| 类型 | 说明 |
|------|------|
| `user/text` | 候选人文本回答 |
| `control/submit_code` | 提交代码（含 language 字段） |
| `control/end_session` | 用户主动结束面试 |
| `ping/pong` | 心跳（30s 间隔） |

**会话管理：**
- Redis Hash 结构：`interview:{id}:state` — stage, questions_asked, chat_history_summary, last_active, status
- ws_token 映射：`ws_token:{token}` → interview_id（TTL 300s）
- 会话 TTL 15 分钟，支持断线重连（Phase 3 完善）
- Redis 不可用时优雅降级（仅在内存运行）

## 第三阶段：评分、报告与语音集成 ✅ 完成

### Phase 3.1 — 多维评分引擎

**创建的文件：**
- `app/agents/scoring_agent.py` — 五维度评分 Agent（技术深度/广度/工程化思维/沟通逻辑/项目经验匹配度）
- `app/services/scoring_service.py` — 评分服务（单题评分 + 维度汇总 + answer_record 构建）

**核心能力：**
- 五维度评分：90-100 优秀 / 70-85 良好 / 60-69 及格 / 0-59 不及格
- 错误分类：fact_error（事实错误）、depth_insufficient（深度不足）
- 实时评分：每道题回答后立即调用 ScoringAgent 评分，结果写入 interviews.answers JSON
- `aggregate_dimensions()` 汇总所有题目分数，计算各维度平均分和通过判定

### Phase 3.2 — 报告生成

**创建的文件：**
- `app/agents/report_agent.py` — 报告生成 Agent
- `app/services/report_generator.py` — 报告生成服务

**核心能力：**
- 综合所有评分生成最终报告（阶段总结、简历扣分、改进建议）
- 简历扣分范围 0-15 分（特殊严重可到 20 分），基于岗位匹配度
- 报告数据写入 reports 表，同步更新 interviews.overall_score/passed
- 面试结束（正常结束/手动结束/超时）始终生成报告

### Phase 3.3 — 联网搜索

**创建的文件：**
- `app/services/web_search.py` — 联网搜索服务

**核心能力：**

- `search(query)` — 返回标题/摘要/链接
- `verify_knowledge(claim)` — 验证不确定知识点，结果标注来源供评分参考

### Phase 3.4 — 音频组件

## 故障排查

### MySQL Access denied 错误

**错误现象：**
```
sqlalchemy.exc.OperationalError: (1045, "Access denied for user 'root'@'localhost' (using password: YES)")
```

**可能原因及排查：**

1. **`.env` 密码与 MySQL 不一致** — 最常见
   ```bash
   # 查看 .env 中的密码
   cat .env | grep DB_PASSWORD
   # 用 mysql 命令行验证
   mysql -u root -p -h 127.0.0.1
   ```

2. **启动目录不对，没读到 `.env`**
   - `pydantic-settings` 默认从当前工作目录读取 `.env`
   - 确保在项目根目录（`.env` 所在目录）启动 uvicorn
   - 或设置环境变量 `ENV_FILE=backend/.env`

3. **MySQL 用户认证插件不兼容**
   - `aiomysql` / `pymysql` 需要 `mysql_native_password` 插件
   - 检查：`SELECT user, host, plugin FROM mysql.user WHERE user='root';`
   - 如果不是 `mysql_native_password`，需修改：
     ```sql
     ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'your_password';
     FLUSH PRIVILEGES;
     ```

4. **MySQL 服务未启动**
   ```bash
   # Windows
   net start MySQL80
   # Docker
   docker-compose up -d mysql
   ```

**当前环境确认（2026-04-27）：**
- `.env` 中 `DB_PASSWORD=zhangbo`，`DB_USER=root`
- MySQL root@localhost 使用 `mysql_native_password` 插件 ✅
- 连接测试通过 ✅
