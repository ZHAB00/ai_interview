## 新增/强化模块详解

### 1. 统一日志系统 (`core/logging_config.py`)

- **目的**：所有请求、Agent 调用、错误都携带 `request_id` 或 `session_id`，便于串联排查。

- **实现**：使用 Python `logging` 库，定义 JSON Formatter，在中间件中生成 `request_id`，存入 `ContextVar`，日志自动注入。

- **示例关键点**：
  
  ```python
  import logging
  import json
  from contextvars import ContextVar
  
  request_id_ctx: ContextVar[str] = ContextVar('request_id', default='-')
  
  class JSONFormatter(logging.Formatter):
      def format(self, record):
          record.request_id = request_id_ctx.get()
          return json.dumps(record.__dict__, ensure_ascii=False)
  ```

- **使用**：`main.py` 启动时调用 `logging_setup()` 初始化。

### 2. 中间件显式化 (`core/middleware.py`)

- **CORS 配置**：定义允许的源、方法、凭据，方便切换开发/生产。
- **限流**：采用 `slowapi`，对登录、注册接口限流（如每分钟 5 次）。
- **集成**：`main.py` 中 `app.add_middleware()` 按需添加。

### 3. 数据库会话健壮性 (`api/deps.py`)

- **关键**：使用 `yield` 的依赖必须正确关闭会话。
  
  ```python
  async def get_db():
      async with SessionLocal() as session:
          yield session
          await session.close()  # 显式关闭，或依赖 context manager
  ```

- **建议**：`SessionLocal` 使用 `AsyncSession`（SQLAlchemy 1.4+），所有服务层函数接收 `session` 参数。

- **Alembic 配置提醒**：`alembic/env.py` 必须导入所有模型的 `Base.metadata`，否则迁移会遗漏表。

### 4. WebSocket 会话状态结构化 (`ws/session_manager.py`)

- **存储设计（Redis Hash）**：
  - 键名：`interview:{interview_id}:state`
  - 字段：
    - `stage`：当前阶段名
    - `questions_asked`：已问题目 ID 列表（JSON）
    - `current_question_index`：当前题目指针
    - `chat_history_summary`：对话摘要（JSON）
    - `last_active`：最后活跃时间戳
- **操作**：
  - 每次状态变更调用 `redis.hset()`
  - 心跳更新 `last_active`
  - TTL 设为 15 分钟，断线后可在期限内恢复

## 给DeepSeek V4 Pro的执行提示

- **Agent 提示词**：请全部使用中文，符合模拟面试官的语气，且输出 JSON 时严格按 Schema 要求。
- **日志**：在每个服务函数和 Agent 调用开始时，使用 `logging.info(f"session={session_id} - ...")` 输出关键步骤。
- **数据库**：所有 ORM 操作使用 `AsyncSession`，查询用 `select()` 语法，关联字段用 `relationship` 预加载。
- **音频标准**：PCM 16kHz 16bit mono，`audio_handler` 接收原始字节流，内部编码转换（如需要）。
- **状态码**：严格遵循接口文档中的错误码约定，抛出 `HTTPException` 时附带自定义 `code`。
- **测试**：每个接口至少有一个正向流程测试和一个验证失败测试。
