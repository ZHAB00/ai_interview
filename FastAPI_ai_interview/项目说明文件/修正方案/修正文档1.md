## 修订内容详情

### 修订1：服务端音频存储职责

**更新 `后端开发说明.md` — `audio_handler.py` 描述**：

```python
# audio_handler.py 职责：
# 1. speech_to_text(audio_bytes) → str：转发云STT，返回识别文本
# 2. text_to_speech(text) → bytes：调用云TTS，返回音频字节
# 3. append_chunk(interview_id, audio_bytes)：追加音频帧到临时文件
# 4. merge_and_upload(interview_id) → str：合并临时文件为完整WAV，上传对象存储，返回URL
```

**更新 `后端开发说明.md` — `interview_orchestrator.py` 面试结束逻辑**：

> 面试结束时（收到 `control/end_session` 或超时触发），先调用 `audio_handler.merge_and_upload()` 获取录音URL，再写入 `interviews.answers` 的各条目 `user_audio_url` 字段，最后发送 `session/end`。

### 修订4：后端超时兜底

**更新 `后端开发说明.md` — `interview_orchestrator.py`**：

> 面试开始时记录 `started_at`，每次收到消息时调用 `check_timeout()` 方法。若当前时间 - `started_at` > 2700秒（45分钟），主动发送 `session/end`，标记 `status='completed'`，关闭 WebSocket。

**更新 `.env.example`**：

```
INTERVIEW_MAX_DURATION=2700
```

### 修订6：WebSocket Token 重连

**更新 `接口文档v1.0.md` §6.5**：

> ws_token 有效期 5 分钟，用于首次连接。后端在首次连接成功后，将 token 与 interview_id 的映射存入 Redis，有效期 15 分钟。断线重连时，若原 token 已过期但 Redis 映射存在且 interview 状态为 `in_progress`，允许重连。重连时使用原 ws_token，服务端通过 Redis 映射校验。

**更新 `后端开发说明.md` — `session_manager.py`**：

> 增加字段：`ws_token_valid` → 记录 token 对应的 interview_id，TTL 15 分钟。重连校验时先查此映射。
