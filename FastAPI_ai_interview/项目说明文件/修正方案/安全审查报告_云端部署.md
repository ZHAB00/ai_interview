## 二、关键发现 (Critical)

### C1. 真实 API 密钥明文暴露 ⚠️ 需手动操作

**位置：** `FastAPI_ai_interview/.env`

以下真实凭证明文存储在项目目录中：

| 行号 | 配置项 | 原始值 |
|------|--------|--------|
| 5 | DB_PASSWORD | 弱密码 `***` |
| 9 | SECRET_KEY | 可猜测的默认值 `***` |
| 15 | DEEPSEEK_API_KEY | 已泄露，需轮换 `***` |
| 49 | DASHSCOPE_API_KEY | 已泄露，需轮换 `***` |
| 65 | ALIYUN_ACCESS_KEY_ID | 已泄露，需轮换 `***` |
| 66 | ALIYUN_ACCESS_KEY_SECRET | 已泄露，需轮换 `***` |
| 67 | ALIYUN_NLS_APPKEY | 已泄露，需轮换 `***` |

**风险：** 任何有文件系统访问权限的人可窃取全部云服务凭证。DeepSeek API Key 可被盗用以产生大额 LLM 调用费用。阿里云 AccessKey 对可被用于 SMS 发送、语音识别等服务的恶意调用。

**已完成：** `.gitignore` 已创建，`.env` 已排除出 Git 管理范围。  
**待手动：** 去 DeepSeek 和阿里云控制台「API Key 管理」页面，删除旧 Key，重新生成填入 `.env`。

### C3. 上传目录完全公开无认证 ✅ 已修复

**位置：** `app/main.py:60-65`、`app/services/vector_store.py:16`

**原问题：**
```python
app.mount("/uploads", StaticFiles(directory=str(upload_dir)), name="uploads")
_VECTOR_DIR = Path(settings.UPLOAD_DIR) / "faiss"
```

**修复内容：**
1. FAISS 向量库从 `uploads/faiss/` 移至独立的 `data/faiss/`，配置键 `FAISS_DIR`
2. `/uploads` 不再整目录挂载，仅挂载 `audio` 和 `documents` 子目录
3. `resumes/` 子目录不再通过 URL 访问

**变更文件：** `main.py`, `vector_store.py`, `config.py`, `.env`, `.env.example`, `Dockerfile`

---

### C4. JWT 密钥可猜测 ⚠️ 需手动操作

**位置：** `.env:9`、`app/core/config.py:29`

```
SECRET_KEY=dev-secret-key-do-not-use-in-production  # .env 当前值
SECRET_KEY: str = "your-secret-key-change-in-production"  # config.py 默认值
```

**风险：** 攻击者可用此密钥伪造任意 JWT Token（任意 user_id、任意 role 含 admin），完全接管系统。

**注意：** `config.py` 默认值保留弱值作为 fallback，强制开发者配置真实密钥。  
**待手动（高优先级）：** `openssl rand -hex 32` 生成强密钥，替换 `.env` 中的 SECRET_KEY。

---

### C5. 前端 Token 存储于 localStorage ⬜ 未修复

**位置：** `vue_ai_interview/src/stores/authStore.js:7-9, 18-19, 58-60`

**风险：** localStorage 可被同源任意 JS 读取，XSS 漏洞可致 Token 泄露。

**暂时缓解：** CSP 头部已添加（M3），Source Map 已关闭（H8），Vue DevTools 已条件加载（L4）——降低 XSS 风险。  
**待修复：** 迁移至 HttpOnly Cookie，需要前后端同步改造，评估为较大工程，列入 P2。

## 三、高风险 (High)

### H1. 数据库密码多处硬编码 ✅ 已修复

**位置：** `docker-compose.yml`、`alembic.ini`

**修复内容：**
- `alembic.ini`：移除硬编码 `mysql+pymysql://root:root@...`，改为空值，由 `env.py` 运行时从 settings 读取
- `docker-compose.yml`：所有密码改为 `${MYSQL_ROOT_PASSWORD:?required}` 等环境变量引用
- MySQL 端口绑定改为 `127.0.0.1:3306:3306`

---

### H2. Redis 无密码认证 + 端口暴露 ✅ 已修复

**位置：** `docker-compose.yml`、`app/core/config.py`

**修复内容：**
- `config.py`：新增 `REDIS_PASSWORD` 配置项，`REDIS_URL` 属性支持密码认证
- `docker-compose.yml`：Redis 容器添加 `requirepass`，端口绑定改为 `127.0.0.1:6379:6379`
- `.env` / `.env.example`：新增 `REDIS_PASSWORD=` 配置键

---

### H3. DEBUG=true 开启 SQL 回显 ✅ 已修复

**位置：** `app/core/database.py:8`、`app/core/config.py`

**修复内容：**
- `config.py`：新增 `DB_ECHO: bool = False` 独立配置项
- `database.py`：`echo=settings.DEBUG` 改为 `echo=settings.DB_ECHO`
- `.env` / `.env.example`：新增 `DB_ECHO=false`

---

### H4. 速率限制全局禁用 ⚠️ 需手动操作

**位置：** `.env:35` — `RATE_LIMIT_ENABLED=false`

**风险：** 可暴力破解登录，刷 SMS 产生费用，耗尽服务器资源。

**待手动：** 本地开发可保持 false，部署时在服务器 `.env` 中设为 `RATE_LIMIT_ENABLED=true`。

---

### H5. Docker 容器以 root 运行 ✅ 已修复

**位置：** `Dockerfile`

**修复内容：** 添加非 root 用户 `appuser`，并设置目录权限：
```dockerfile
RUN groupadd -r appuser && useradd -r -g appuser appuser && \
    chown -R appuser:appuser /app
USER appuser
```

---

### H6. 全栈无 HTTPS/TLS ⬜ 未修复

**位置：** `docker-compose.yml`、`Dockerfile`、`vite.config.js`

整个通信链路明文 HTTP/Ws。  
**待修复：** 部署时在服务器前置 Nginx 反向代理 + Let's Encrypt TLS 终止，配置 `X-Forwarded-Proto` 头。此修复属于基础设施层，不涉及应用代码。

---

### H7. Flower 监控面板无认证暴露 ✅ 已修复

**位置：** `docker-compose.yml`

**修复内容：**
- Flower 命令添加 `--basic_auth=${FLOWER_USER:-admin}:${FLOWER_PASSWORD:?required}`
- 端口绑定改为 `127.0.0.1:5555:5555`

---

### H8. 前端生产构建含 Source Map ✅ 已修复

**位置：** `vue_ai_interview/vite.config.js`

**修复内容：** 添加生产构建配置：
```javascript
build: { sourcemap: false }
```
