# AI 面试官 — API 接口文档 v1.0

## 1. 概述

* **基础 URL**：`https://api.example.com`

* **通信方式**：REST API（业务操作）+ WebSocket（实时面试对话）

* **认证方式**：
  
  * REST API：在请求头中携带 `Authorization: Bearer {access_token}`
  * WebSocket：连接时通过查询参数传递临时 token `?token={ws_token}`

* **通用请求头**：
  
      Content-Type: application/json
      Authorization: Bearer <access_token>

* **通用响应格式**：
  
  * 成功：返回对应 JSON 数据
  
  * 错误：
    
        {
          "error": {
            "code": "ERROR_CODE",
            "message": "人类可读的错误描述"
          }
        }

* **分页**：列表接口统一参数 `page` (默认1)、`page_size` (默认20，最大100)，响应体包含 `items`、`total`、`page`、`page_size`。

* **时间格式**：所有时间字段均为 ISO 8601 UTC 格式，如 `2026-04-26T10:00:00Z`。

* **角色枚举**：`candidate`（求职者）、`admin`（管理员/运营方）。

## 2. 用户认证模块 `/api/auth`

### 2.1 用户注册

**POST** `/api/auth/register`

* **说明**：手机号作为唯一标识，同时需提供用户名和密码。

* **请求体**：
  
      {
        "phone": "13800138000",
        "username": "zhangsan",
        "password": "123456"
      }

* **响应** `201 Created`：
  
      {
        "user_id": "u_abc123",
        "phone": "13800138000",
        "username": "zhangsan",
        "role": "candidate",
        "created_at": "2026-04-26T10:00:00Z"
      }

* **错误**：`409 Conflict`（手机号已注册）、`422 Unprocessable Entity`（参数校验失败）。

### 2.2 用户登录

**POST** `/api/auth/login`

* **说明**：使用手机号+密码登录，返回 JWT 令牌对。

* **请求体**：
  
      {
        "phone": "13800138000",
        "password": "123456"
      }

* **响应** `200 OK`：
  
      {
        "access_token": "eyJhbGciOi...",
        "refresh_token": "eyJhbGciOi...",
        "token_type": "Bearer",
        "expires_in": 86400
      }

* **错误**：`401 Unauthorized`（账号或密码错误）。

### 2.3 刷新令牌

**POST** `/api/auth/refresh`

* **说明**：使用 `refresh_token` 获取新的 `access_token` 和 `refresh_token`。
* **请求头**：`Authorization: Bearer {refresh_token}`
* **响应** `200 OK`：同登录响应结构。
* **错误**：`401 Unauthorized`（token 无效或过期）。

## 3. 简历模块 `/api/resumes`

### 3.1 上传并解析简历

**POST** `/api/resumes/upload`

* **说明**：上传 PDF/Word/图片格式简历，**同步解析**（等待解析完成再返回），同时传入目标岗位用于计算岗位匹配度。简历扣分仅在最终报告中体现，此处不返回扣分。

* **请求**：`multipart/form-data`
  
  * `file`：简历文件（必填，支持 pdf, docx, jpg, png，最大 10MB）
  * `position`：目标岗位，如 "AI Agent开发工程师"（必填）
  * `difficulty`：难度，可选 `初级/中级/高级`，默认 `中级`

* **响应** `200 OK`：
  
      {
        "resume_id": "r_xyz123",
        "parsed_data": {
          "name": "张三",
          "phone": "13800138000",
          "email": "zhangsan@example.com",
          "education": [
            { "school": "某某大学", "degree": "本科", "major": "计算机科学", "year": "2024" }
          ],
          "skills": ["Python", "FastAPI", "LangChain"],
          "work_experience": [
            { "company": "XX科技", "position": "后端开发", "duration": "2024-至今", "description": "..." }
          ],
          "project_experience": [ ... ]
        },
        "position_match_score": 75,
        "match_feedback": "技能匹配度较高，但缺乏指定的Agent开发项目经验"
      }

* **错误**：`422`（文件格式不支持）、`413`（文件过大）。

### 3.2 获取简历详情

**GET** `/api/resumes/{resume_id}`

* **响应** `200 OK`：结构同 `parsed_data`。

## 4. 面试管理模块 `/api/interviews`

### 4.1 创建面试并获取 WebSocket 凭证

**POST** `/api/interviews`

* **说明**：基于已解析的简历创建一次模拟面试，返回必需的 `ws_token` 用于建立实时通信。

* **请求体**：
  
      {
        "resume_id": "r_xyz123",
        "position": "AI Agent开发工程师",
        "difficulty": "中级",
        "mode": "full",
        "selected_stages": ["初筛","HR面","技术面"]
      }
  
  * `mode`：`full`（全流程串行）或 `stage`（任选阶段练习）
  * 当 `mode=stage` 时，必须提供 `selected_stages` 数组，至少包含一个阶段

* **响应** `201 Created`：
  
      {
        "interview_id": "iv_abc001",
        "ws_token": "temp_token_XyZ123",
        "ws_url": "wss://api.example.com/ws/interview/iv_abc001",
        "expires_in": 300
      }
  
  `ws_token` 有效期 5 分钟，仅用于该次面试的 WebSocket 连接。

### 4.2 获取面试历史列表

**GET** `/api/interviews/history?page=1&page_size=20`

* **说明**：获取当前登录用户的所有面试记录。

* **响应** `200 OK`：
  
      {
        "items": [
          {
            "interview_id": "iv_abc001",
            "position": "AI Agent开发工程师",
            "difficulty": "中级",
            "mode": "full",
            "status": "completed",
            "overall_score": 82,
            "passed": true,
            "created_at": "2026-04-26T09:00:00Z"
          }
        ],
        "total": 1,
        "page": 1,
        "page_size": 20
      }
  
  状态枚举：`created`、`in_progress`、`completed`、`abandoned`。

### 4.3 获取最终面试报告

**GET** `/api/interviews/{interview_id}/report`

* **说明**：面试结束后调用。报告生成需要一定时间（通常 10-30 秒），若尚未完成，前端可每隔 5 秒轮询一次。

* **响应**：
  
  * 生成中 `200 OK`：
    
        {
          "status": "generating"
        }
  
  * 完成 `200 OK`：
    
        {
          "status": "completed",
          "report": {
            "overall_score": 82,
            "passed": true,
            "threshold": 60,
            "dimensions": {
              "技术深度": 85,
              "技术广度": 78,
              "工程化思维": 80,
              "沟通逻辑": 90,
              "项目经验匹配度": 75
            },
            "stage_breakdown": [
              {
                "stage": "技术面",
                "questions": [
                  {
                    "question_text": "请解释TCP三次握手",
                    "user_answer_summary": "三次握手就是...",
                    "score": 85,
                    "dimensions_scores": { "技术深度": 85, "沟通逻辑": 80 },
                    "strengths": "基本流程正确",
                    "weaknesses": "缺少对为什么是三次的深入解释",
                    "errors": [
                      {
                        "type": "fact_error",
                        "snippet": "TCP是三次握手，不是四次",
                        "correction": "表述正确但过于简陋，未展开步骤和状态变化",
                        "suggestion": "建议依次描述 SYN、SYN-ACK、ACK 三个步骤及状态变化。"
                      }
                    ]
                  }
                ]
              }
            ],
            "resume_deduction": -5,
            "deduction_reason": "简历技能中缺乏Agent相关项目经验，与目标岗位存在一定偏差",
            "overall_comment": "整体表现良好，技术基础扎实，但在系统设计类问题上深度不足...",
            "generated_at": "2026-04-26T12:00:00Z"
          }
        }

* **错误**：`404`（面试不存在或不属于当前用户）。

## 5. 管理后台接口 `/api/admin`

所有管理接口需登录且角色为 `admin`，中间件校验。

### 5.1 题库管理 `/api/admin/questions`

#### 5.1.1 创建题目

**POST** `/api/admin/questions`

* **说明**：创建结构化题目，包含完整评分要素。

* **请求体**：
  
      {
        "stage": "技术面",
        "position_tags": ["AI Agent开发工程师", "后端开发"],
        "difficulty": "中级",
        "type": "technical",
        "question_text": "请解释Python的GIL锁及其对多线程的影响",
        "dimensions": ["技术深度", "技术广度"],
        "scoring_points": [
          { "point": "正确描述GIL是什么", "max_score": 20 },
          { "point": "解释GIL对CPU密集型任务的影响", "max_score": 40 },
          { "point": "列举绕过GIL的方法", "max_score": 40 }
        ],
        "sample_answer": "GIL即全局解释器锁...",
        "follow_up_hints": ["追问：GIL是否影响IO密集型任务？", "追问：Python多进程如何替代多线程"],
        "tags": ["Python", "GIL"]
      }
  
  * `type` 枚举：`technical`（技术题）、`behavioral`（行为题）、`situational`（情境题）

* **响应** `201 Created`：
  
      { "question_id": "q_tech_001", "created_at": "..." }

#### 5.1.2 查询题目列表

**GET** `/api/admin/questions?stage=技术面&position=AI%20Agent&difficulty=中级&page=1&page_size=20`

* **响应**：分页列表，每项包含题目完整信息。

#### 5.1.3 更新题目

**PUT** `/api/admin/questions/{question_id}`

* **请求体**：同创建，所有字段可选（部分更新）。
* **响应** `200 OK`。

#### 5.1.4 删除题目

**DELETE** `/api/admin/questions/{question_id}`

* **响应** `204 No Content`。

### 5.2 非结构化文档管理 `/api/admin/documents`

#### 5.2.1 上传文档

**POST** `/api/admin/documents`

* **说明**：上传 PDF/Word/文本等文档，用于补充知识库。上传后异步切片+向量化，存入 FAISS。

* **请求**：`multipart/form-data`
  
  * `file`：文档文件（必填）
  * `description`：文档描述（可选）
  * `tags`：标签（可选，逗号分隔）

* **响应** `202 Accepted`：
  
      {
        "document_id": "doc_456",
        "status": "processing",
        "message": "文档已上传，正在向量化处理，请稍后查询状态"
      }

#### 5.2.2 查询文档处理状态

**GET** `/api/admin/documents/{document_id}`

* **响应** `200 OK`：
  
      {
        "document_id": "doc_456",
        "filename": "company_interview_guide.pdf",
        "description": "公司面试规范",
        "status": "ready",
        "chunks_count": 150,
        "created_at": "2026-04-26T11:00:00Z"
      }
  
  `status` 可能值：`processing`、`ready`、`error`。

#### 5.2.3 删除文档

**DELETE** `/api/admin/documents/{document_id}`

* **说明**：同时删除 FAISS 向量索引和数据库记录。
* **响应** `204 No Content`。

#### 5.2.4 重新处理文档

**POST** `/api/admin/documents/{id}/reprocess`

* **说明**：对处于 `error` 状态的文档重新触发向量化处理。
* **响应** `202 Accepted`：
  ```json
  { "document_id": 456, "status": "processing", "message": "文档已重新处理" }
  ```
* **错误**：`422`（文档状态非 error）。

### 5.3 用户管理 `/api/admin/users`（可选，初期可简化）

* **GET** `/api/admin/users?page=&role=` 列表
* **PUT** `/api/admin/users/{user_id}/disable` 禁用用户等

### 5.5 反馈举报 `/api/feedback`

#### POST `/api/feedback/{interview_id}`

* **说明**：提交对评分错误或评分公正性的反馈/纠正。
* **请求体**：
  ```json
  {
    "question_index": 0,
    "error_index": 0,
    "feedback_type": "fact_error",
    "comment": "我认为AI判断有误..."
  }
  ```
  * `feedback_type`：`fact_error`（事实错误）/ `score_unfair`（评分不公）/ `other`（其他）
* **响应** `201 Created`：
  ```json
  { "status": "recorded", "feedback_id": 1 }
  ```
* **错误**：`404`（面试不存在或不属于当前用户）。

## 6. WebSocket 实时通信协议

### 6.1 连接地址

`wss://api.example.com/ws/interview/{interview_id}?token={ws_token}`

### 6.2 消息格式约定

* 所有非音频消息使用 **JSON 文本帧**。
* 音频消息使用 **二进制帧**（直接发送 PCM 或 WAV 字节流）。

### 6.3 消息类型速查

#### 服务端 → 客户端

| 类型 `type`                  | 说明         | 载荷 `data` 示例                                                                                      |
| -------------------------- | ---------- | ------------------------------------------------------------------------------------------------- |
| `session/started`          | 面试会话正式开始   | `{"interview_id":"...","stage":"初筛","message":"欢迎参加面试"}`                                          |
| `ai/status`                | AI状态变化     | `{"status":"thinking"}` / `listening` / `speaking`                                                |
| `ai/text`                  | AI回复文本（流式） | `{"text":"你好，首先请做一下自我介绍","is_final":false}`                                                       |
| `ai/audio`                 | AI语音流（二进制） | 原始音频字节，边接收边播放                                                                                     |
| `control/stage_change`     | 阶段切换通知     | `{"from":"HR面","to":"技术面","message":"接下来进入技术面试环节"}`                                               |
| `control/coding_challenge` | 弹出编程题      | `{"question_id":"q_code_01","title":"实现快速排序","language":"python","description":"请实现一个快速排序算法..."}` |
| `ai/code_review_error`     | 代码评审失败     | `{"question_id":"...","error":"评审超时，请重试"}`                                                         |
| `session/end`              | 面试结束       | `{"reason":"completed","message":"面试已结束，报告生成中"}`                                                  |
| `pong`                     | 心跳回复       | `{}`                                                                                              |

#### 客户端 → 服务端

| 类型 `type`             | 说明         | 载荷                                                                                |
| --------------------- | ---------- | --------------------------------------------------------------------------------- |
| `user/audio`          | 用户音频流      | 二进制帧，实时麦克风数据                                                                      |
| `user/text`           | 手动输入文本（备用） | `{"text":"我可以重新说一遍吗"}`                                                            |
| `user/interrupt`      | 用户主动打断     | `{}`（VAD 检测到说话时自动发送，或点击打断按钮发送）                                                    |
| `control/submit_code` | 提交编程题代码    | `{"question_id":"q_code_01","code":"def quicksort(arr):...","language":"python"}` |
| `control/end_session` | 用户提前结束面试   | `{}`                                                                              |
| `ping`                | 心跳         | `{}`                                                                              |

### 6.4 关键交互流程

**1) 建立连接**

1. 客户端用 `ws_token` 连接，服务端校验通过后发送 `session/started`，携带面试上下文。

**2) 语音对话主循环**

* 用户说话时，前端持续发送 `user/audio` 二进制帧。
* 服务端接收后实时转写，并在整句话完成后调用面试官 Agent 生成回复。
* 服务端发送 `ai/text`（流式）和 `ai/audio`（二进制）给客户端。
* 播放期间若收到 `user/interrupt`（或自动 VAD 信号），服务端立即停止 TTS，清空缓冲，进入聆听状态。

**3) 编程题交互**

* 服务端发送 `control/coding_challenge`，前端弹出编辑器。
* 用户编写代码并提交，前端发送 `control/submit_code`。
* 服务端调用代码评审 Agent，将评审结果通过 `ai/text` 发送（同时可能发送 `ai/audio` 读出来），然后回归正常对话流程。

**4) 阶段切换**

* 当面试官 Agent 判断当前阶段问题问完，服务端发送 `control/stage_change`，前端显示过渡动画。

**5) 面试结束**

* 自然结束或用户主动发送 `control/end_session`（不区分结束类型，始终生成报告）。
* 服务端发送 `session/end`，关闭 WebSocket。
* 前端转而去轮询 `GET /interviews/{id}/report` 获取报告。
* 后端超时兜底：面试开始后 45 分钟（2700 秒）自动结束面试。

### 6.5 断线重连

* 客户端检测 WebSocket 断开后，若 `ws_token` 未过期且面试状态仍在 `in_progress`，可尝试重连。
* ws_token 首次连接成功后 TTL 延长至 15 分钟，与 session TTL 一致。
* 重连时携带相同 token，服务端通过 Redis 映射校验，恢复会话状态（当前阶段、已问题目数）。
* 重连后服务端重发 `session/started` 和 `control/stage_change` 消息恢复上下文。

## 7. 错误码参考

| HTTP 状态码 | 错误代码 `code`           | 说明                      |
| -------- | --------------------- | ----------------------- |
| 400      | `BAD_REQUEST`         | 请求格式错误                  |
| 401      | `UNAUTHORIZED`        | 未登录或 token 无效           |
| 403      | `FORBIDDEN`           | 权限不足（如非管理员访问后台）         |
| 404      | `NOT_FOUND`           | 资源不存在                   |
| 409      | `CONFLICT`            | 资源冲突（如手机号已注册）           |
| 413      | `FILE_TOO_LARGE`      | 上传文件超过大小限制              |
| 422      | `VALIDATION_ERROR`    | 参数校验失败，`details` 字段包含详情 |
| 429      | `RATE_LIMITED`        | 请求过于频繁                  |
| 500      | `INTERNAL_ERROR`      | 服务器内部错误                 |
| 503      | `SERVICE_UNAVAILABLE` | 服务暂不可用（如报告生成中）          |

### WebSocket 关闭码

| 关闭码  | 说明                                |
| ------ | --------------------------------- |
| 4001   | `WS_UNAUTHORIZED` — token 无效或已过期 |
| 4004   | 关联简历不存在                          |
