## 一、服务器初始化

### 1.1 安装 Docker（国内服务器）

官方脚本被墙，国内改用 Ubuntu apt 源直接安装：

```bash
sudo apt update
sudo apt install docker.io docker-compose-v2 -y
sudo usermod -aG docker $USER
newgrp docker
```

### 1.2 配置 Docker 镜像加速

国内 Docker Hub 直连超时，必须配镜像源。以下源实测可用（2026-05）：

```bash
sudo tee /etc/docker/daemon.json << 'EOF'
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://docker.registry.cyou",
    "https://mirror.baidubce.com",
    "https://docker.nju.edu.cn"
  ]
}
EOF
sudo systemctl restart docker
docker --version
```

> **踩坑：** 以下源已失效 — 网易 hub-mirror.c.163.com、中科大 ustc、阿里云公共加速器、dockerhub.icu、dockerproxy.com。多配置几个备用源，Docker 会自动选择最快且可用的。

### 1.3 安装工具链

```bash
sudo apt install git certbot nginx -y
```

## 二、代码拉取 & 环境变量

### 2.1 克隆项目

```bash
git clone https://github.com/ZHAB00/ai_interview.git /opt/ai_interview
cd /opt/ai_interview/FastAPI_ai_interview
cp .env.example .env
```

### 2.2 编辑 .env

```bash
nano .env
```

必须修改的配置项：

```env
DB_PASSWORD=你的数据库密码
DB_USER=app_user            # 不能用 root
SECRET_KEY=<openssl rand -hex 32 的结果>
DEEPSEEK_API_KEY=sk-你的key
DASHSCOPE_API_KEY=sk-你的key
REDIS_PASSWORD=你的Redis密码
DEBUG=false
RATE_LIMIT_ENABLED=true
CORS_ORIGINS=https://你的域名
```

### 2.3 补充 Docker 专用变量

`docker-compose.yml` 用 `${VAR:?required}` 强制校验，Docker Compose 默认读 `.env` 文件。追加缺失变量：

```bash
cat >> .env << 'EOF'
MYSQL_ROOT_PASSWORD=你的数据库密码
FLOWER_USER=admin
FLOWER_PASSWORD=随机密码
EOF
```

> **踩坑：** 如果 `.env` 里缺 `MYSQL_ROOT_PASSWORD`、`FLOWER_USER`、`FLOWER_PASSWORD`，Docker Compose 会报 `required variable is missing a value`。同时 `DB_USER` 不能设为 `root`（MySQL 容器保留字），否则 MySQL 容器 exit(1)。

## 四、前端构建 & 部署

### 4.1 安装 Node.js

```bash
# 如果之前有残留 Docker 源干扰，先清理
sudo rm -f /etc/apt/sources.list.d/docker.list

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install nodejs -y
```

### 4.2 构建前端

```bash
cd /opt/ai_interview/vue_ai_interview
npm install
```

> **踩坑：** Vite 8 默认不包含 esbuild，`npm run build` 报 `Cannot find package 'esbuild'`。需手动安装：
> ```bash
> npm install esbuild --save-dev
> ```

### 4.3 配置生产环境 API 地址 & 构建

```bash
echo 'VITE_API_BASE_URL=https://你的域名' > .env.production
npm run build
```

> **踩坑：** 不设 `VITE_API_BASE_URL`，前端 API 请求会发到空地址，页面空白。

### 4.4 部署到 Nginx

```bash
sudo mkdir -p /var/www/ai_interview
sudo cp -r dist/* /var/www/ai_interview/
```

## 五、Nginx + HTTPS 配置

### 5.1 先配 HTTP（证书还没签发时）

```bash
sudo tee /etc/nginx/sites-available/ai_interview << 'EOF'
server {
    listen 80;
    server_name 你的域名;

    location /.well-known/acme-challenge/ {
        root /var/www/ai_interview;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 120s;
    }

    location /ws/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 3600s;
        proxy_buffering off;
    }

    location /uploads/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
    }

    location / {
        root /var/www/ai_interview;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}
EOF

sudo ln -s /etc/nginx/sites-available/ai_interview /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t && sudo nginx -s reload
```

### 5.2 签发 SSL 证书

```bash
sudo mkdir -p /var/www/ai_interview/.well-known/acme-challenge
sudo certbot certonly --webroot -w /var/www/ai_interview -d 你的域名
```

> **踩坑：** certbot 验证时报 404，因为 Nginx 配置里缺少 `/.well-known/acme-challenge/` 路由。必须先加好。

### 5.3 启用 HTTPS

```bash
sudo tee /etc/nginx/sites-available/ai_interview << 'EOF'
server {
    listen 80;
    server_name 你的域名;

    location /.well-known/acme-challenge/ {
        root /var/www/ai_interview;
    }

    location / {
        return 301 https://$host$request_uri;
    }
}

server {
    listen 443 ssl http2;
    server_name 你的域名;

    ssl_certificate /etc/letsencrypt/live/你的域名/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/你的域名/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_session_cache shared:SSL:10m;

    add_header Strict-Transport-Security "max-age=63072000" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Frame-Options "DENY" always;

    client_max_body_size 20m;

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
    }

    location /ws/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 3600s;
        proxy_buffering off;
    }

    location /uploads/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
    }

    location / {
        root /var/www/ai_interview;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}
EOF

sudo nginx -t && sudo nginx -s reload
```

### 5.4 证书自动续期

```bash
echo '0 3 * * * certbot renew --quiet && systemctl reload nginx' | sudo crontab -
```

## 八、踩坑录

| # | 问题 | 原因 | 解决 |
|---|------|------|------|
| 1 | Docker 安装脚本被墙 | 国内网络 | `apt install docker.io` |
| 2 | Docker Hub 拉镜像超时 | 直连被墙 | 配国内镜像源（daocloud/registry.cyou/baidubce/nju） |
| 3 | Docker daemon 启动失败 | `daemon.json` 含不可见字符/BOM | 用 `printf` 或 `tee` 重写，勿复制粘贴多行文本 |
| 4 | `MYSQL_USER="root"` 导致 MySQL 崩溃 | MySQL 官方镜像保留字 | `DB_USER=app_user` |
| 5 | Docker Compose 报 `required variable is missing` | `.env` 文件每行前有空格/Docker Compose 同时读多个 env 文件 | 空格删除；所有变量统一放到 `.env` |
| 6 | certbot 验证 404 | Nginx 未配置 `.well-known` 路由 | 添加 `location /.well-known/acme-challenge/` |
| 7 | `npm run build` 失败 | Vite 8 不包含 esbuild | `npm install esbuild --save-dev` |
| 8 | 前端页面空白 | 未配 `VITE_API_BASE_URL` | 创建 `.env.production` 填入后端域名 |
| 9 | GitHub git pull/push 超时 | HTTP/2 故障 | `git config --global http.version HTTP/1.1` |
| 10 | 短信验证码发送后验证失败 | Redis SCAN 扫到非 JSON 值导致 `int` 类型错误 | `isinstance(item, dict)` 防御检查 |
| 11 | 后端启动报 pydantic `extra_forbidden` | `.env` 有 Docker 专用变量但 Settings 模型不认识 | `config.py` 加 `extra="ignore"` |
| 12 | 面试录音文件无法保存 `Audio merge failed: name 'user_id' is not defined` | `merge_and_upload` 未接收 `user_id` 参数，但内部调用 `_upload` 时用了 `user_id` | `merge_and_upload` 加 `user_id` 参数；`end_interview` 传递该参数 |
| 13 | 每次克隆仓库后要手动 `mkdir -p uploads/{audio,documents,resumes} && chmod 777` | 宿主机 `./uploads` 绑定挂载导致 Docker appuser 无写入权限 | docker-compose 改用命名卷 `uploads_data:/app/uploads`，Docker 自动管权限 |

## 九、常用运维命令

```bash

docker compose ps

docker compose exec backend bash
```
