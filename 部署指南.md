# 部署指南 — AI面试官

## 前置条件

- 云服务器：2核 4GiB 以上，安装 Ubuntu 22.04 / 24.04
- 一个域名，DNS 已解析到服务器 IP
- 端口 80 / 443 可外网访问（安全组/防火墙）

---

## 一、服务器初始化

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com | sudo bash
sudo usermod -aG docker $USER
newgrp docker

# 安装 Docker Compose
sudo apt install docker-compose-plugin -y

# 安装 certbot（用于免费 HTTPS 证书）
sudo apt install certbot -y

# 安装辅助工具
sudo apt install git curl vim -y
```

---

## 二、拉取代码 & 配置环境变量

```bash
# 克隆项目到服务器（或用 scp 上传）
git clone <你的仓库地址> /opt/ai_interview
cd /opt/ai_interview/FastAPI_ai_interview

# 复制环境变量模板
cp .env.example .env
```

### 编辑 `.env`，填入生产配置

```bash
vim .env
```

**必须修改的项目：**

```env
# 数据库
DB_PASSWORD=<生成随机密码>
# 生产环境建专用用户，不用 root
DB_USER=app_user

# JWT — 最重要
SECRET_KEY=<openssl rand -hex 32 的结果>

# API Keys — 去对应平台申请
DEEPSEEK_API_KEY=sk-你的key
DASHSCOPE_API_KEY=sk-你的key

# Redis
REDIS_PASSWORD=<生成随机密码>

# 生产环境
DEBUG=false
RATE_LIMIT_ENABLED=true

# CORS — 生产环境只允许你的域名
CORS_ORIGINS=https://你的域名.com

# SMS（可选，不配则开发模式打印验证码到日志）
ALIYUN_ACCESS_KEY_ID=
ALIYUN_ACCESS_KEY_SECRET=
ALIYUN_SMS_SIGN_NAME=
ALIYUN_SMS_TEMPLATE_REGISTER=
```

## 三、创建数据库专用用户（非 root）

`docker-compose.yml` 已配置创建专用用户。首次启动后确认：

```bash
# MySQL 用 root 连进去
docker exec -it ai_interview-mysql-1 mysql -uroot -p"${MYSQL_ROOT_PASSWORD}"

# 创建应用用户并授权
CREATE USER IF NOT EXISTS 'app_user'@'%' IDENTIFIED BY '<DB_PASSWORD>';
GRANT SELECT, INSERT, UPDATE, DELETE ON ai_interview.* TO 'app_user'@'%';
FLUSH PRIVILEGES;
```

## 四、配置 Nginx + HTTPS

### 4.1 复制 Nginx 配置

```bash
sudo mkdir -p /var/www/certbot

# 编辑 nginx.conf，替换 YOUR_DOMAIN 为实际域名
sudo cp nginx.conf /etc/nginx/sites-available/ai_interview
sudo ln -s /etc/nginx/sites-available/ai_interview /etc/nginx/sites-enabled/
sudo nginx -t && sudo nginx -s reload
```

### 4.2 签发 SSL 证书

```bash
sudo certbot certonly --webroot -w /var/www/certbot -d 你的域名.com
```

### 4.3 启用 HTTPS

编辑 `/etc/nginx/sites-available/ai_interview`，取消 SSL 相关行的注释：

```nginx
ssl_certificate     /etc/letsencrypt/live/YOUR_DOMAIN/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/YOUR_DOMAIN/privkey.pem;
ssl_trusted_certificate /etc/letsencrypt/live/YOUR_DOMAIN/chain.pem;
```

```bash
sudo nginx -t && sudo nginx -s reload
```

### 4.4 自动续期

```bash
# certbot 自带 systemd timer，确认已激活
sudo systemctl status certbot.timer

# 测试续期
sudo certbot renew --dry-run
```

## 五、构建并启动应用

### 5.1 设置 Docker 环境变量

```bash
# 创建 docker-compose 需要的环境变量文件
cat > .env.docker << 'EOF'
MYSQL_ROOT_PASSWORD=<生成随机密码>
DB_USER=app_user
DB_PASSWORD=<同上.env里的DB_PASSWORD>
REDIS_PASSWORD=<同上.env里的REDIS_PASSWORD>
SECRET_KEY=<同上.env里的SECRET_KEY>
FLOWER_USER=admin
FLOWER_PASSWORD=<生成随机密码>
DEEPSEEK_API_KEY=<同上.env>
EOF
```

### 5.2 启动服务

```bash
# 构建镜像（首次较久，需下载依赖和嵌入模型 ~400MB）
docker compose --env-file .env.docker build

# 启动所有服务
docker compose --env-file .env.docker up -d

# 等待就绪后检查状态
docker compose ps
```

服务列表：

| 服务 | 端口 | 说明 |
|------|------|------|
| mysql | 127.0.0.1:3306 | 数据库，仅本地可连 |
| redis | 127.0.0.1:6379 | 缓存/队列，仅本地可连 |
| backend | 8000 | FastAPI，通过 Nginx 反代 |
| celery-worker | — | 异步文档向量化 |
| flower | 127.0.0.1:5555 | Celery 监控，仅本地可连 |

### 5.3 数据库初始化

```bash
# 创建表结构
docker compose exec backend alembic upgrade head

# 预置面试题（可选）
docker compose exec backend python -m app.main --seed
```

### 5.4 验证

```bash
# 后端健康检查
curl http://localhost:8000/api/health

# 通过 Nginx
curl https://你的域名.com/api/health
```

## 六、构建前端

```bash
cd /opt/ai_interview/vue_ai_interview

# 安装 Node.js（如未装）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install nodejs -y

# 安装依赖 & 构建
npm install
npm run build

# 部署到 Nginx 静态目录
sudo mkdir -p /var/www/ai_interview
sudo cp -r dist/* /var/www/ai_interview/
```

然后编辑 `/etc/nginx/sites-available/ai_interview`，取消前端静态文件部分的注释：

```nginx
location / {
    root /var/www/ai_interview;
    index index.html;
    try_files $uri $uri/ /index.html;

    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

```bash
sudo nginx -t && sudo nginx -s reload
```

## 七、创建管理员账号

```bash
# 先通过注册页面注册一个普通用户，然后进 MySQL 提升权限
docker compose exec mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" -e \
  "UPDATE ai_interview.users SET role='admin' WHERE phone='你的手机号';"
```

重新登录后，导航栏会自动出现邀请码入口和题库/文档管理。

## 九、常用运维命令

```bash
# 查看所有日志
docker compose logs -f --tail=100

# 单独看后端日志
docker compose logs -f backend --tail=100

# 重启某个服务
docker compose restart backend

# 运行数据库迁移
docker compose exec backend alembic upgrade head

# 进入容器调试
docker compose exec backend bash

# 停止所有服务
docker compose down

# 完全重建（数据不动，容器重建）
docker compose up -d --build
```

## 十、常见问题

### 启动后 502

检查：`docker compose ps` 看 backend 是否 healthy；`docker compose logs backend` 看是否连不上数据库。

### TTS / ASR 无声音

检查 `.env` 中 `DASHSCOPE_API_KEY` 是否已配置并正确。日志中搜索 `Qwen TTS bridge` 和 `Qwen ASR bridge`。

### 嵌入模型下载超时

Celery Worker 启动日志会显示 `Model loaded: BAAI/bge-small-zh-v1.5`。若超时重试几次，模型缓存后就不再下载。

### 磁盘空间

嵌入模型约 400MB + Docker 镜像约 2GB。监控 `uploads/audio/` 大小（每次面试产生 WAV 文件）。
