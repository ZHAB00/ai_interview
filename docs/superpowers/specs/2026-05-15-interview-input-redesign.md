# 面试输入区重设计

**日期**: 2026-05-15
**范围**: 桌面端 + 手机端 InterviewRoom.vue 输入区

## 问题

当前输入区三个问题：
1. **输入框太小** — 单行 `el-input`，`size="small"`，宽 220px，无法舒适地打字回答面试问题
2. **布局不合理** — 语音麦克风（左）和文字输入（右）分居 footer 两端，文字输入被挤在角落像备胎
3. **交互反馈不足** — 发送无状态提示，无字数统计，"正在回答"的感觉缺失

## 方案：统一输入栏

将语音和文字融合到一条输入栏中（类似 ChatGPT/微信风格），替代当前 footer 左右分立的布局。

### 布局结构

```
[快捷回复 chip]  ← 仅 AI 等待时显示
┌─────────────────────────────────────────┐
│  🎤  │  textarea（自动扩展）  │  发送  │
└─────────────────────────────────────────┘
```

- 麦克风按钮在左侧（圆形，36px 桌面 / 30px 手机）
- textarea 中间自适应填充剩余空间
- 发送按钮在右侧

### 桌面端 (≥768px)

| 属性 | 值 |
|------|-----|
| textarea 初始高度 | 2 行 (~60px) |
| textarea 最大高度 | ~150px，超出内部滚动 |
| 字号 | 14px |
| placeholder | "输入你的回答，或点击麦克风语音输入..." |
| 快捷回复 chip | flex-wrap 换行 |
| 提示文字 | 右下角 "Enter 发送 · Shift+Enter 换行"（10px 灰色） |

### 手机端 (<768px)

| 属性 | 值 |
|------|-----|
| textarea 初始高度 | 1 行 (~36px) |
| textarea 最大高度 | ~100px |
| 字号 | 13px（防止 iOS 自动缩放） |
| placeholder | "输入回答..." |
| 快捷回复 chip | 横向滚动，不换行 |
| footer 布局 | **不换行**，所有元素一行 |

### 移动端布局原则

- 统一输入栏始终一行，不换行
- 元素间距和尺寸等比缩小，保持可用性

## 交互行为

### Enter 键
- **Enter** → 发送消息
- **Shift+Enter** → 换行

### 语音/文字互斥
- 点击麦克风开始录音 → 文字输入区锁定（灰底 + "文字输入已锁定"）
- 文字输入区获得焦点/有内容 → 自动取消录音
- 同一时间只有一种输入方式活跃
- 录音为**点击切换**（非长按），点击开始/再次点击停止，最长 58 秒

### 发送反馈
- 点击发送 → 按钮变为 loading 动画（spinner），输入框整体灰色禁用
- placeholder 变为 "正在发送..."
- AI 开始回复后 → 输入栏保持禁用，placeholder 变为 "面试官正在回复..."

### AI 状态联动
- AI 说话/生成时：整个输入栏灰色禁用（opacity 0.6）
- 麦克风按钮 disabled + 灰色
- textarea disabled，placeholder = "面试官正在回复..."
- 发送按钮 disabled，显示 spinner + "回复中"
- 快捷回复 chip 隐藏
- AI 说完后：恢复可用状态

### 快捷回复
- 位置：输入栏上方
- 显示时机：AI 等待用户回答时（非录音中、非 AI 说话中）
- 内容：2-3 个 chip 标签，如"可以再说一遍吗""请换一种问法""跳过这个问题"
- 交互：点击 chip → 填入 textarea，不自动发送
- 数据来源：`constants.js` 中定义默认列表

## 实现涉及文件

| 文件 | 改动 |
|------|------|
| `InterviewRoom.vue` | 重写 footer 模板和样式，新增 textarea、chip、状态联动逻辑 |
| `useInterview.js` | 可能需要新增 `sendTextMessage` 方法，调整互斥逻辑 |
| `constants.js` | 新增 `QUICK_REPLIES` 常量 |

## 不变的部分

- WebSocket 通信逻辑
- 语音录制/解析流程（仅调整 UI 层互斥逻辑）
- 消息展示/对话区域
- AIStatusIndicator、AudioWaveform 组件（保留但可能调整位置）
- 草稿气泡（draft bubble）编辑流程
- StageTransition、CountdownTimer、CodeEditorPanel

## 不做

- 智能占位文字（动态变化）
- 字数统计显示
- 草稿 localStorage 保留
- AI 说完自动聚焦
- 即时清空（发送后保留文字，实际行为取决于互斥逻辑）
