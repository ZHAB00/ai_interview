输入区细节打磨

布局定了，这些交互细节你哪些想要？（多选）

⌨ 键盘交互

✓

Enter 直接发送

按 Enter 发送消息，Shift+Enter 换行。面试场景下更流畅，不用鼠标点发送按钮

✓

Ctrl+Enter 发送

Enter 换行，Ctrl+Enter 发送。更适合需要分段组织回答的场景

🎤 语音 vs 文字切换

✓

无缝切换

正在语音输入时，点输入框打字不会打断录音，两者可以互补（比如录音后文字微调）

✓

互斥模式

开始打字 → 取消录音；开始录音 → 清空文字。每次只用一种方式，逻辑更简单清晰

📎 发送反馈

✓

发送动画 + 状态

点击发送后按钮变 loading/对勾，输入框短暂禁用。让用户明确知道"已发送，正在等 AI 回复"

✓

即时清空

发送后立刻清空输入框，焦点保留，可以直接输入下一个回答（适合快节奏对话）

🔒 状态联动

✓

AI 说话时禁用输入

面试官正在说话/生成回答时，输入框灰色禁用，防止打断。面试礼仪的一部分

✓

始终可输入

即使 AI 在说话也能提前输入，消息会在 AI 说完后自动发送。减少等待感

✓

AI 说完自动聚焦

面试官说完话后，输入框自动获得焦点，用户可以立刻打字，不用点一下

✨ 小功能

✓

草稿保留

未发送的内容在切换页面/刷新后仍然保留（localStorage），不会丢字

✓

快捷回复建议

输入框上方出现 2-3 个建议回复的 chip 标签，点击即可填入（比如"请换一种问法""可以再说一遍吗"）
