### AI面试官 — 项目需求分析文档

## 1. 项目概述

**产品定位**：面向求职者的AI模拟面试练习平台。由平台运营方维护核心知识库，同时允许求职者自定义题库。系统通过解析简历，基于岗位和难度自动匹配知识库，进行多阶段、实时语音的模拟面试，并提供能力画像、评分和错误解析报告。

**目标用户**：

- **求职者**：上传简历，进行模拟面试练习，查看评分报告。
- **平台管理员/运营方**：维护题库、管理用户、查看面试记录。

**核心价值**：低成本、随时随地模拟大厂面试全流程，获得客观的能力评估和精准的提升建议。

---

## 2. 用户核心流程

1. **配置与开始**：注册/登录 → 上传简历（PDF/Word/图片） → 选择“面试类型 + 岗位 + 难度” → 选择面试模式（全流程串行/任选单阶段） → 开始面试。
2. **面试中**：与AI进行实时语音对话，AI根据简历和题库动态提问、追问、切换阶段。技术面时，在提示下打开在线代码编辑器完成编程题。
3. **面试后**：查看面试记录（录音+逐字稿）、综合评分报告（能力画像、总评）、错误解析（高亮+正确示例）。

---

## 3. 前端呈现需求

### 3.1 全局与导航

- **适配**：仅适配PC Web端，为后续桌面应用打包（Electron/Tauri）做准备。
- **布局**：采用清晰的三明治或侧边栏布局。
- **状态可见性**：始终显示用户登录状态、面试进行状态、网络连接状态。

### 3.2 面试前：首页与配置面板

- **简历上传**：
  - 支持拖拽或点击上传 PDF、Word、图片格式。
  - 上传后显示解析状态（解析中/完成/失败）。
  - **简历扣分非实时**：上传后不即时显示扣分，仅在最终报告中体现。
- **面试配置**：
  - 岗位选择器：下拉或搜索，预置岗位（如“AI Agent开发工程师”）。
  - 难度选择器：初级/中级/高级。
  - 模式选择器：单选按钮“全流程面试”或“阶段练习”。
  - *条件显示*：若选“阶段练习”，则出现阶段多选框（如：初筛、HR面、技术一面、技术二面、终面）。
- **引导与状态**：首次使用有引导提示。展示预置的示例简历和岗位供快速体验。

### 3.3 面试中：核心交互界面

这是系统的核心界面，需具备高度沉浸感。

- **摄像头画面**：启用摄像头，将画面作为背景或小窗展示，仅采集，不加特效或分析。
- **AI面试官形象（可选）**：一个动态的AI头像或波形动画，跟随AI说话状态跳动。
- **对话状态**：
  - **聆听态**：麦克风图标激活，有波形律动，AI形象安静。
  - **思考态**：显示“对方正在输入...”或加载动画。
  - **说话态**：AI形象跳动，播放TTS流式音频。
  - **打断态**：用户开口说话，AI音频被立即中断，迅速回到聆听态。
  - **静音态**：编程题时，麦克风自动静音，显示静音图标。
- **语音交互（PTT/连续）**：采用连续对话模式，用户自然开口即可打断。
- **阶段过渡**：全流程模式下，弹出过渡提示卡片（如“第一阶段（初筛）已完成，即将进入HR面”）。
- **在线代码编辑器集成**：
  - **触发**：AI语音提示后，编辑器区域从右侧/底部平滑弹出。
  - **界面**：嵌入Monaco Editor，支持语法高亮，上方显示题目描述。
  - **操作**：用户编写代码。完成后点击“提交并评审”按钮。
  - **反馈**：提交后编辑器变为只读，并在下方/侧边显示AI评审意见（文本）。点击“继续面试”，编辑器收起。
- **应急操作**：清晰的“结束面试”按钮，点击后需二次确认。

### 3.4 面试后：报告与回放

- **报告面板**：
  - **能力画像**：雷达图展示五大维度（技术深度/广度、工程化思维、沟通逻辑、项目经验匹配度）。各维度得分、总分、通过/不通过判定（单项低于阈值直接不通过）。
  - **综合评语**：AI生成的总体评价和分阶段点评。
  - **错误解析**：列表形式展示“事实错误”和“深度不足”两类问题，每条附高亮定位和优化示例。
- **面试回放**：
  - **同步播放器**：界面同时显示语音播放条和滚动高亮的逐字稿，点击文稿可跳转对应音频。
- **交互**：错误解析项旁有“举报/纠正”按钮。

## 4. 后端呈现需求（API与逻辑）

### 4.1 API设计风格

采用 RESTful API + WebSocket 混合模式。

- **REST API**：处理业务逻辑（简历上传、配置、历史记录、题库管理）。
- **WebSocket**：维持面试实时会话，负责文本/音频流、控制指令（打断、切换阶段）。

### 4.2 核心API模块

- **用户认证模块**：
  - `POST /api/auth/register`、`POST /api/auth/login`
  - 返回JWT令牌，后续请求需在Header携带。
- **简历模块**：
  - `POST /api/resumes/upload`：上传文件，后端解析并存储。
  - `GET /api/resumes/{id}`：获取解析后的简历结构数据。
- **面试管理模块**：
  - `POST /api/interviews/start`：创建面试实例。接收`resume_id`、`position`（岗位）、`difficulty`（难度）、`mode`（模式）等参数。返回`interview_id`和WebSocket连接Token。
  - `GET /api/interviews/{id}/report`：获取最终报告。
  - `GET /api/interviews/history`：获取历史记录列表。
- **知识库管理模块（管理员）**：
  - `POST /api/admin/questions`：创建题目（复杂结构，含评分要点JSON）。
  - `POST /api/admin/documents`：上传非结构化文档，触发FAISS索引构建。
  - `GET/PUT/DELETE` 相应接口。

### 4.3 WebSocket实时通信协议

这是系统的核心数据流，所有消息均为JSON格式。

- **握手与初始化**：
  - 客户端连接：`ws://host/ws/interview/{interview_id}?token=...`
  - 服务端验证后发送 `{"type": "session/started", "data": {"interview_id": ..., "stage": "初筛"}}`
- **音频与识别流（客户端→服务端）**：
  - `{"type": "audio/data", "data": "Base64编码的音频块"}`
  - 服务端实时转发至云STT，得到识别文本。
- **用户文本（客户端→服务端）**：
  - `{"type": "user/text", "data": {"text": "识别出的或手动输入的文本", "timestamp": ...}}`
- **VAD打断指令（客户端→服务端）**：
  - `{"type": "user/interrupt"}`
  - 服务端立即中断当前TTS流，清空待播放队列，准备聆听。
- **AI回复（服务端→客户端）**：
  - `{"type": "ai/status", "data": {"status": "thinking"}}`
  - `{"type": "ai/text", "data": {"text": "LLM回复文本", "is_final": false}}` (流式文本)
  - `{"type": "ai/audio", "data": "TTS音频块Base64"}` (随后播放)
  - `{"type": "ai/status", "data": {"status": "listening"}}`（TTS播放完毕）
- **控制指令（服务端→客户端）**：
  - `{"type": "control/stage_change", "data": {"from": "HR面", "to": "技术面"}}`
  - `{"type": "control/coding_challenge", "data": {"question": "实现一个快速排序...", "language": "Python"}}`：触发前端打开编辑器。
  - `{"type": "control/coding_submit_response", "data": {"review": "AI评审意见..."}}`
  - `{"type": "session/end", "data": {"interview_id": ...}}`：会话结束。

### 4.4 核心逻辑处理器

- **面试编排器**：
  - **流程控制**：维护一个状态机，管理阶段切换、题目分发、追问逻辑。
  - **提示词工程**：通过高度封装的提示词，让LLM扮演面试官，结合简历、检索到的题目、背景文档、实时回答，动态生成问题。
- **RAG检索器**：
  - **混合检索**：结构化题目走PostgreSQL标签检索；非结构化知识走FAISS向量检索。
  - **查询生成**：根据当前阶段和关键词生成查询。
- **评分引擎**：
  - **维度评分**：根据你的量表（60/70-85/90-100）对每个能力维度打分。
  - **错误分析器**：调用LLM对比回答与题库要点，识别事实错误和深度不足。
- **联网搜索Agent**：
  - 异步独立Agent，仅在评分遇到不确定知识时触发，将结果返回补充报告。

## 5. 非功能性需求

- **性能**：语音识别延迟<500ms，打断响应<200ms，页面加载<3s。
- **可用性**：关键操作有确认，报告提供举报/纠正入口。
- **安全性**：API必须有JWT鉴权，实时流使用临时Token，数据库密码加密存储。
- **成本控制**：单次面试时长限制45分钟；前端做静音裁剪节省STT流量；优先使用高性价比模型。

## 7. 潜在风险与应对策略

- **语音识别不准**：允许用户手动修正转写文本，评分时参考修正稿。
- **AI评分主观性**：输出详细评分理由，提供“举报/纠正”入口。
- **前端VAD误触发**：提供灵敏度调节和手动打断按钮作为后备。
- **联网搜索信息噪声**：结果仅作参考标注，不直接修改评分。
- **知识库质量**：管理员需审核核心题库；RAG检索时过滤低质来源。
- **WebSocket断线**：通过Redis维护会话状态，实现断线重连和进度恢复。
- **伦理声明**：系统内显著位置标注“此评分仅供模拟练习参考，不构成真实招聘决策依据”。
