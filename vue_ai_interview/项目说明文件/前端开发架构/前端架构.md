## 1. 技术选型确认

| 层面        | 技术                           | 说明        |
| --------- | ---------------------------- | --------- |
| 框架        | Vue 3 + Composition API      | 单文件组件     |
| 语言        | JavaScript                   | ES6+      |
| 构建        | Vite                         | 快速 HMR    |
| UI 库      | Element Plus                 | 统一风格      |

## 3. 路由设计

| 路径                     | 组件             | 权限  | 说明               |
| ---------------------- | -------------- | --- | ---------------- |
| `/login`               | LoginPage      | 未登录 | 手机号+密码登录         |
| `/register`            | RegisterPage   | 未登录 | 注册               |
| `/dashboard`           | DashboardPage  | 已登录 | 简历上传、面试配置、历史列表入口 |
| `/interview/:id`       | InterviewRoom  | 已登录 | **核心面试页**        |

## 4. 全局组件说明

### 4.1 AppLayout.vue

- **Header**：Logo + 用户头像下拉菜单（个人中心/退出）+ 当前面试状态（面试中显示阶段名）+ 网络状态指示灯（`NetworkStatus`）
- **Content**：`<router-view />`
- **Footer**：固定底部，显示伦理声明：“*此评分为模拟练习参考，不构成真实招聘决策依据*”

### 4.2 NetworkStatus.vue

- WebSocket 连接状态：
  - 绿色圆点 + “已连接”
  - 黄色圆点 + “重连中...”
  - 红色圆点 + “已断开”
- 通过 `interviewStore.wsStatus` 驱动

### 4.3 CountdownTimer.vue

- 接收 `totalSeconds`（默认 2700 = 45 分钟）
- 显示 `MM:SS` 格式
- 最后 5 分钟变红闪烁
- 最后 2 分钟弹出 `el-alert` 警告：“面试即将结束，请尽快完成作答”
- 归零后进入 1 分钟缓冲，缓冲结束强制调用 `endSession()`

### 4.4 StageTransition.vue

- 接收 `from` 和 `to` 阶段名
- 全屏半透明遮罩 + 居中卡片 + CSS 飞入动画
- 内容：“第一阶段（{from}）已完成，即将进入 {to}”
- 3 秒自动消失，或点击“继续”立即关闭

### 4.5 GuideCard.vue

- 首次登录（`localStorage` 无 `guide_shown` 标记）弹出
- 简明图文介绍流程：上传简历 → 配置面试 → 开始对话 → 查看报告
- “知道了”按钮关闭并设置标记，不再显示

## 5. 各页面详细设计

### 5.1 DashboardPage（首页/配置面板）

**布局**：左右两栏或单列居中卡片。

**简历上传区域**：

- `el-upload` 组件，支持拖拽 + 点击
- 文件类型限制：`.pdf,.docx,.jpg,.png`
- 上传后：
  - 进度条展示上传进度
  - **全屏遮罩 + 加载动画 + “正在解析简历，请稍候...”**
  - 解析完成自动消失，显示解析结果摘要（姓名、技能标签等）
  - 解析失败显示红色提示 + “重新上传”按钮

**面试配置区域**（解析完成后显示）：

- 岗位选择：`el-select` 带搜索，预置 “AI Agent开发工程师”
- 难度选择：`el-radio-group`（初级/中级/高级）
- 模式选择：`el-radio-group`（全流程面试 / 阶段练习）
- 条件显示：`v-if="mode === 'stage'"` 展示阶段多选框（初筛、HR面、技术面、终面）
- “开始面试”按钮：调用 `POST /interviews`，获取 `ws_token`，跳转 `/interview/:id`

**历史列表入口**：

- 下方卡片展示最近 5 条面试记录（`el-table` 简化版）
- 点击跳转对应报告页

**快速体验**：

- 卡片：“使用示例简历快速体验”按钮，自动填入预置简历和配置，直接开始

### 5.2 InterviewRoom（面试核心页）

**布局（方案A）**：

```
┌──────────────────────────────────────────────┐
│ Header: 阶段名 | 倒计时 | 网络状态 | 结束按钮   │
├──────────────────────────────┬───────────────┤
│                              │ 摄像头小窗     │
│   对话文本主区域              │ (200×150)     │
│   (逐字稿 + AI回复流式显示)   │ 右上角悬浮     │
│                              │               │
│  [用户]：我熟悉Python...     │ 可拖拽         │
│  [AI]：请解释GIL锁...       │               │
│                              │               │
├──────────────────────────────┴───────────────┤
│ Footer: 麦克风按钮 | 打断按钮 | AI状态 | 手动输入 │
└──────────────────────────────────────────────┘
```

**对话状态机**：

| 状态  | 麦克风图标 | AI 形象            | 说明         |
| --- | ----- | ---------------- | ---------- |
| 聆听态 | 绿色脉动  | 静止               | 等待用户说话     |
| 思考态 | 静音灰色  | “AI 正在思考...”加载动画 | 服务端处理中     |
| 说话态 | 静音灰色  | 波形跳动 + TTS 播放    | AI 回复中     |
| 打断态 | 绿色脉动  | 立刻变暗             | 用户开口，AI 中断 |
| 静音态 | 红色斜杠  | 保持上态             | 编程题时自动静音   |

**Footer 控制栏**：

- 麦克风按钮：点击切换静音/取消静音
- **打断按钮**：手动打断 AI 说话（VAD 失灵时的后备），发送 `user/interrupt`
- **手动输入**：`el-input` + 发送按钮，发送 `user/text`
- AI 状态指示灯（`AIStatusIndicator`）
- **结束面试**：`el-popconfirm` 二次确认：“确定结束面试吗？结束后不可恢复”

**编程编辑器（CodeEditorPanel）**：

- 收到 `control/coding_challenge` 时从右侧滑入
- 宽度占右侧 40-50%，摄像头小窗自动隐藏
- 编辑器上方：题目描述（`el-card`）
- 中部：Monaco Editor，根据 `language` 切换语法高亮
- 下方：“提交并评审”按钮（`el-button`，loading 态）
- 提交后：
  - 编辑器变为只读
  - 下方/侧边显示 AI 评审文本（`ai/text` 流式接收）
  - 若评审失败：显示“评审失败，请重试”+ 重试按钮
- “继续面试”按钮：关闭编辑器，麦克风取消静音

**倒计时**：

- `CountdownTimer` 组件挂载在 Header
- 最后 2 分钟警告，归零后 1 分钟缓冲，缓冲结束强制 `endSession()`

**断线重连**：

- WebSocket 断开时，`NetworkStatus` 变红
- 自动尝试重连 3 次，间隔 1s/3s/5s
- 重连成功恢复状态，失败显示“连接已断开，请刷新页面重试”

## 6. 核心 Composable 设计

### 6.1 useAudioRecorder.js

**流程**：

1. `navigator.mediaDevices.getUserMedia({ audio: true })`
2. `AudioContext` 16kHz 单声道
3. `AudioWorklet` 获取 PCM 原始数据
4. 加载 Silero VAD（`onnxruntime-web`），每帧推理
5. 检测到说话开始 → `ws.send({ type: 'user/interrupt' })` + 开始发送 `user/audio` 二进制帧
6. 检测到说话结束 → 停止发送音频帧
7. VAD 灵敏度可通过变量调节

### 6.2 useWebSocket.js

**核心逻辑**：

```javascript
connect(wsUrl, token):
  ws = new WebSocket(`${wsUrl}?token=${token}`)
  ws.binaryType = 'arraybuffer'

  ws.onmessage = (event) => {
    if (event.data instanceof ArrayBuffer) {
      // 播放 AI 音频
      playAudioChunk(event.data)
    } else {
      const msg = JSON.parse(event.data)
      dispatch(msg.type, msg)
    }
  }

  // 心跳 15 秒
  setInterval(() => ws.send(JSON.stringify({ type: 'ping' })), 15000)

  // 断线重连：最多 3 次
  ws.onclose = () => { if (retryCount < 3) reconnect() }
```

### 6.3 useInterview.js

**状态**：

```javascript
const currentStage = ref('初筛')
const dialogue = ref([])          // { role, text, time, audioUrl? }
const aiStatus = ref('listening') // listening | thinking | speaking
const codingChallenge = ref(null) // { questionId, title, language, description }
const wsStatus = ref('connecting')
const isMicMuted = ref(false)
```

**消息分发**：

- `ai/text` → 流式追加到 `dialogue` 最后一条 AI 消息
- `ai/status` → 更新 `aiStatus`
- `control/stage_change` → 更新 `currentStage`，显示 `StageTransition`
- `control/coding_challenge` → 设置 `codingChallenge`，弹出编辑器
- `session/end` → 跳转 `/report/:interviewId`

## 9. 与服务端接口对接

| 前端操作    | 请求   | 接口                                            |
| ------- | ---- | --------------------------------------------- |
| 注册      | POST | `/api/auth/register`                          |
| 登录      | POST | `/api/auth/login`                             |
| 刷新令牌    | POST | `/api/auth/refresh`                           |
| 上传简历    | POST | `/api/resumes/upload`                         |
| 创建面试    | POST | `/api/interviews`                             |
| 面试历史    | GET  | `/api/interviews/history`                     |
| 获取报告    | GET  | `/api/interviews/{id}/report`（轮询）             |
| 题库 CRUD | *    | `/api/admin/questions`                        |
| 文档管理    | *    | `/api/admin/documents`                        |
| 实时通信    | WS   | `wss://host/ws/interview/{id}?token=ws_token` |
| 举报反馈    | POST | `/api/reports/{id}/feedback`（预留）              |

**Axios 拦截器**：请求自动带 token，401 时自动刷新，刷新失败跳登录。
