# AI面试官 — 数据库表结构设计 vFinal

## 设计决策回顾

| 决策项  | 采用方案                      |
| ---- | ------------------------- |
| 主键   | 自增整数 `INT AUTO_INCREMENT` |
| 简历数据 | 大 JSON 字段                 |
| 面试回答 | 大 JSON 存 interviews 表     |
| 题库删除 | 软删除 `is_deleted`          |
| 报告存储 | 独立 `reports` 表            |
| 用户题库 | MVP 先不做，`created_by` 字段预留 |
| 引擎   | InnoDB，字符集 utf8mb4        |

---

## 1. 用户表 `users`

| 字段              | 类型                        | 约束                                  | 说明         |
| --------------- | ------------------------- | ----------------------------------- | ---------- |
| `id`            | INT                       | PK, AUTO_INCREMENT                  | 用户ID       |
| `phone`         | VARCHAR(20)               | UNIQUE, NOT NULL                    | 手机号，登录唯一标识 |
| `username`      | VARCHAR(50)               | NOT NULL                            | 用户名，2-30字符 |
| `password_hash` | VARCHAR(255)              | NOT NULL                            | bcrypt 哈希  |
| `role`          | ENUM('candidate','admin') | NOT NULL, DEFAULT 'candidate'       | 角色         |
| `created_at`    | DATETIME                  | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 注册时间       |
| `updated_at`    | DATETIME                  | ON UPDATE CURRENT_TIMESTAMP         | 更新时间       |

**索引**：

- `UNIQUE idx_phone` (phone)
- `INDEX idx_role` (role)

---

## 2. 简历表 `resumes`

| 字段                     | 类型                   | 约束                                  | 说明          |
| ---------------------- | -------------------- | ----------------------------------- | ----------- |
| `id`                   | INT                  | PK, AUTO_INCREMENT                  | 简历ID        |
| `user_id`              | INT                  | FK → users.id, NOT NULL             | 所属用户        |
| `file_path`            | VARCHAR(500)         | NOT NULL                            | 原始文件存储路径    |
| `parsed_data`          | JSON                 | NOT NULL                            | 解析后结构化数据    |
| `position`             | VARCHAR(100)         | NOT NULL                            | 目标岗位        |
| `difficulty`           | ENUM('初级','中级','高级') | NOT NULL, DEFAULT '中级'              | 难度          |
| `position_match_score` | INT                  | DEFAULT NULL                        | 岗位匹配度 0-100 |
| `match_feedback`       | TEXT                 | DEFAULT NULL                        | 匹配度说明文字     |
| `uploaded_at`          | DATETIME             | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 上传时间        |

**`parsed_data` JSON 结构**：

```json
{
  "name": "张三",
  "phone": "13800138000",
  "email": "zhangsan@example.com",
  "education": [
    { "school": "某大学", "degree": "本科", "major": "计算机科学", "year": "2024" }
  ],
  "skills": ["Python", "FastAPI", "LangChain"],
  "work_experience": [
    { "company": "XX科技", "position": "后端开发", "duration": "2024-至今", "description": "..." }
  ],
  "project_experience": [
    { "name": "某项目", "role": "核心开发", "description": "..." }
  ]
}
```

**索引**：

- `INDEX idx_user_id` (user_id)
- `INDEX idx_position` (position)

## 3. 题目表 `questions`

| 字段                | 类型                                           | 约束                                  | 说明               |
| ----------------- | -------------------------------------------- | ----------------------------------- | ---------------- |
| `id`              | INT                                          | PK, AUTO_INCREMENT                  | 题目ID             |
| `stage`           | VARCHAR(50)                                  | NOT NULL                            | 阶段：初筛/HR面/技术面/终面 |
| `position_tags`   | JSON                                         | NOT NULL                            | 适用岗位标签数组         |
| `difficulty`      | ENUM('初级','中级','高级')                         | NOT NULL                            | 难度               |
| `type`            | ENUM('technical','behavioral','situational') | NOT NULL                            | 题型               |
| `question_text`   | TEXT                                         | NOT NULL                            | 题目内容             |
| `dimensions`      | JSON                                         | NOT NULL                            | 考察维度数组           |
| `scoring_points`  | JSON                                         | NOT NULL                            | 评分要点             |
| `sample_answer`   | TEXT                                         | DEFAULT NULL                        | 参考答案             |
| `follow_up_hints` | JSON                                         | DEFAULT NULL                        | 追问方向数组           |
| `tags`            | JSON                                         | DEFAULT NULL                        | 标签数组             |
| `is_deleted`      | TINYINT(1)                                   | NOT NULL, DEFAULT 0                 | 软删除标记            |
| `created_by`      | INT                                          | DEFAULT NULL                        | 创建者ID（NULL=平台）   |
| `created_at`      | DATETIME                                     | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间             |
| `updated_at`      | DATETIME                                     | ON UPDATE CURRENT_TIMESTAMP         | 更新时间             |

**`scoring_points` JSON 结构**：

```json
[
  { "point": "正确描述GIL是什么", "max_score": 20 },
  { "point": "解释GIL对CPU密集型的影响", "max_score": 40 },
  { "point": "列举绕过GIL的方法", "max_score": 40 }
]
```

**索引**：

- `INDEX idx_stage_difficulty` (stage, difficulty)
- `INDEX idx_is_deleted` (is_deleted)
- `INDEX idx_created_by` (created_by)

## 5. 面试表 `interviews`

| 字段                 | 类型                                                    | 约束                                  | 说明                |
| ------------------ | ----------------------------------------------------- | ----------------------------------- | ----------------- |
| `id`               | INT                                                   | PK, AUTO_INCREMENT                  | 面试ID              |
| `user_id`          | INT                                                   | FK → users.id, NOT NULL             | 候选人               |
| `resume_id`        | INT                                                   | FK → resumes.id, NOT NULL           | 关联简历              |

| `difficulty`       | ENUM('初级','中级','高级')                                  | NOT NULL                            | 难度                |
| `mode`             | ENUM('full','stage')                                  | NOT NULL                            | 全流程/阶段练习          |
| `selected_stages`  | JSON                                                  | DEFAULT NULL                        | 阶段列表（mode=stage时） |
| `status`           | ENUM('created','in_progress','completed','abandoned') | NOT NULL, DEFAULT 'created'         | 面试状态              |
| `answers`          | JSON                                                  | DEFAULT NULL                        | 所有问答对             |

## 7. 实体关系图

```
users (1) ────< (N) resumes
  │
  └── (1) ────< (N) interviews (1) ──── (1) reports
                    │
                    └── answers JSON (内嵌问答对)

questions (平台题库，软删除)
documents (非结构化文档)
```

---

## 8. 设计备注

- `interviews.answers` 直接存 JSON，查询逐题数据在应用层解析。
- 题库软删除后，`answers` 中已保存题目文本快照，历史报告不受影响。
- 所有 JSON 字段建议 MySQL ≥ 5.7，若版本低于此需要降为 TEXT。
- 音频文件存对象存储，数据库仅存路径。
- `created_by` 字段已预留，后续可扩展用户自定义题库。
