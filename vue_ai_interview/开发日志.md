## 2026-04-27 — 交互设计逻辑自修复（5项）

审查所有 composable 和组件的数据流、状态同步、条件逻辑，发现并修复以下自相矛盾的逻辑缺陷：

| # | 问题 | 根因 | 修复 |
|---|------|------|------|
| 1 | VAD 灵敏度滑块无效 | `speechThreshold` 是 `const` 在初始化时计算，`setSensitivity()` 只更新了无关联的 `sensitivityLevel` ref | 改为 `computed(() => 0.7 - sensitivityLevel.value * 0.4)`，`detect()` 使用 `.value` |
| 2 | Monaco Editor CDN 配置失效 | 两个 `onMounted` hook：先 `loader.init()` 后 `loader.config()`，配置在初始化之后才生效 | 合并为单个 `onMounted`，`loader.config()` 在 `loader.init()` 之前调用 |
| 3 | 摄像头初始化竞态条件 | `initCamera()` 先获取 stream 再检查 `cameraRef.value`，但 video 元素被 `v-if="cameraEnabled"` 隐藏（初始为 false）导致 ref 为 null | 先设置 `cameraEnabled/showCamera = true` → `await nextTick()` → 再 `cameraRef.value.srcObject = stream` |
| 4 | 编程题提交后缺失只读态 + 麦克风未恢复 | 提交代码后编辑器仍可编辑（设计文档要求只读）；"继续面试"未取消麦克风静音 | 新增 `isCodeSubmitted` ref → 传 `:readOnly` 给 CodeEditorPanel；`handleContinueInterview` 中调用 `toggleMic()` 恢复录音 |
| 5 | 面试房间内重复 NetworkStatus | `AppLayout.vue` 面试期间已显示 NetworkStatus，`InterviewRoom.vue` 又渲染了一个 | 移除 InterviewRoom 中的 `<NetworkStatus />` 及 import |
